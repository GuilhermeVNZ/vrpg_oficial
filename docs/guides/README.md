# Developer Guides

Esta pasta contém guias práticos para desenvolvedores trabalhando no VRPG Client.

## Quando Criar Guias

Guias devem ser criados quando:

1. **Há um processo complexo** que precisa ser documentado passo a passo
2. **Há uma integração específica** que requer instruções detalhadas
3. **Há um workflow de desenvolvimento** que precisa ser padronizado
4. **Há configurações avançadas** que precisam de explicação detalhada

## Estrutura Esperada

### Guias Planejados

- **SETUP.md** - Guia completo de setup do ambiente de desenvolvimento
- **BUILD.md** - Guia de build e compilação do projeto
- **TESTING.md** - Guia detalhado de como executar e escrever testes
- **DEBUGGING.md** - Guia de debugging e troubleshooting
- **CONTRIBUTING.md** - Guia de contribuição para o projeto
- **DEPLOYMENT.md** - Guia de deploy e distribuição (pode referenciar docs/DEPLOYMENT.md)

### Quando Serão Criados

Esses guias serão criados conforme necessário durante o desenvolvimento:

- **Fase 0-1**: SETUP.md, BUILD.md (quando a infraestrutura base estiver pronta)
- **Fase 2-3**: TESTING.md, DEBUGGING.md (quando os testes estiverem implementados)
- **Fase 4-5**: CONTRIBUTING.md (quando o projeto estiver pronto para contribuições)
- **Fase 6+**: Guias específicos conforme necessidade

## Relação com Outros Documentos

- **docs/TESTING.md** - Estratégia geral de testes (alto nível)
- **docs/guides/TESTING.md** - Guia prático de como executar testes (passo a passo)

- **docs/DEPLOYMENT.md** - Visão geral de deploy
- **docs/guides/DEPLOYMENT.md** - Guia passo a passo de deploy
